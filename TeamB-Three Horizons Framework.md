# Three Horizons Framework — PSP AI-Driven Personalised Healthcare Platform

## Core idea

PSP transforms from a pharmacy retailer into a **data-driven, personalized healthcare platform**, using PSP card data, transaction history, and behavioral insights.

The three horizons are **connected through a single data backbone**, where each stage builds capabilities for the next.

---

## The three horizons

| Horizon | Time frame | Focus | Role of data |
|---|---|---|---|
| **H1: Defend and Extend** | Now – 12 months | Use PSP card and transaction data to improve efficiency and personalization | Data collection and basic usage |
| **H2: Nurture and Build** | 12 – 36 months | Integrate and activate data across all channels | Unified customer view and decision systems |
| **H3: Create and Options** | 36+ months | Build AI-driven personalized healthcare ecosystem | Continuous learning and autonomous decisions |

---

## Horizon 1 — Defend and Extend (Data activation)

| Initiative | Description | Data used | Outcome |
|---|---|---|---|
| **PSP Card Personalization Engine** | Use PSP card purchase history to recommend products (e.g. skincare, vitamins, OTC) | Transaction history, frequency, basket composition | Higher conversion, increased basket size |
| **Demand Forecasting by Customer Behavior** | Forecast demand based on PSP card purchasing patterns by location and segment | Store-level + customer-level data | Reduced stockouts, better inventory planning |
| **AI Customer Service (App + Chat)** | Provide answers based on user profile and purchase history | Customer profile + past interactions | Faster service, lower cost |

**What happens here:**  
- PSP starts **using existing data (PSP card)**  
- Data is still fragmented but already creates value  

👉 Output of H1:  
- Cleaned transaction data  
- Basic customer segmentation  
- Early personalization models  

---

## Horizon 2 — Nurture and Build (Data integration)

| Initiative | Description | Data integration | Outcome |
|---|---|---|---|
| **360° Customer Health Profile** | Combine PSP card data, online behavior, and product usage into one profile | Offline + app + web + product data | Full view of customer needs |
| **Dynamic Pricing & Promotion Engine** | Personalized discounts based on customer sensitivity and behavior | Pricing + transaction + behavioral data | Higher margin + targeted promotions |
| **Omnichannel Inventory Optimization** | Allocate stock based on predicted demand per customer segment | Inventory + demand + location data | Higher availability, less waste |

**What happens here:**  
- PSP moves from **data usage → data integration**  
- Systems are connected (stores + app + supply chain)

👉 Output of H2:  
- Unified data platform  
- Real-time decision capability  
- Personalization at scale  

---

## Horizon 3 — Create and Options (AI-driven ecosystem)

| Initiative | Description | Data intelligence | Outcome |
|---|---|---|---|
| **AI Personal Health Assistant** | AI recommends routines (e.g. skincare, supplements) based on history and goals | Full customer profile + behavior + outcomes | Continuous engagement, higher lifetime value |
| **Automated Replenishment System** | AI predicts when customer will need products and suggests or auto-orders | Consumption patterns + purchase cycles | Subscription-like revenue model |
| **Doctor–Influencer Integrated Platform** | Connect recommendations (medical + lifestyle) directly to purchase | External + internal data integration | New platform business model |

**What happens here:**  
- AI becomes:
  - proactive  
  - predictive  
  - partially autonomous  

👉 PSP evolves into:
- **healthcare platform, not just retailer**

## Sequencing logic

| Horizon | Builds on | Enables |
|---|---|---|
| H1 | Existing PSP card + transaction data | Data cleaning + early models |
| H2 | H1 data foundation | Full personalization + system integration |
| H3 | H2 infrastructure | AI-driven healthcare ecosystem |

👉 Key insight:
**Without integrating PSP card + behavioral data in H2, H3 is impossible**

---

## Funding logic

- H1 → generates **quick wins (sales + cost savings)**  
- H2 → funded by H1 gains  
- H3 → funded after:
  - data platform is built  
  - personalization is proven

PSP should:

3. **Use AI to move from reactive to proactive healthcare**

## Key takeaway

The transformation is not about adding AI tools —  
it is about building a **data-driven personalization engine** that evolves into a **healthcare platform**.

Each horizon increases:
- data quality  
- decision intelligence  
- customer lifetime value
